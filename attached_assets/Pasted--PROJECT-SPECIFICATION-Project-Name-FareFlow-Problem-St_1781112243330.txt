# PROJECT SPECIFICATION

## Project Name

**FareFlow* 

---

## Problem Statement

Daily commuters waste time opening multiple ride-hailing apps such as Uber, Ola, Rapido, and Namma Yatri to compare fares before booking a ride.

Users also have no visibility into whether current prices are likely to increase or decrease in the next few minutes due to surge pricing.

FareFlow solves this by aggregating fare estimates into a single dashboard, identifying the cheapest option, predicting future surge conditions, and providing intelligent booking recommendations. 

---

## Project Goal

Build a modern AI-powered ride comparison platform that:

1. Accepts pickup and destination locations.
2. Calculates ride distance.
3. Simulates fares for multiple ride providers.
4. Compares all fares in one dashboard.
5. Highlights the cheapest ride.
6. Predicts future surge pricing.
7. Recommends whether the user should book now or wait.
8. Suggests alternative transport methods.
9. Provides a clean modern user experience suitable for a hackathon demo.

## Fare Simulation Logic

Create separate providers:

Uber

Ola

Rapido

Namma Yatri

---

### Uber Formula

Base Fare = 50

Per Km = 12

Formula:

fare = 50 + distance × 12 + surge

---

### Ola Formula

Base Fare = 45

Per Km = 13

Formula:

fare = 45 + distance × 13 + surge

---

### Rapido Formula

Base Fare = 35

Per Km = 11

Formula:

fare = 35 + distance × 11 + surge

---

### Namma Yatri Formula

Base Fare = 40

Per Km = 10

Formula:

fare = 40 + distance × 10 + surge
